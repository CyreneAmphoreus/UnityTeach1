# Unity 坦克炮管俯仰学习记录（与 GPT 对话整理）

> 本文档整理自一次我与 GPT 关于 Unity 坦克炮管俯仰（Gun Pitch）实现问题的调试对话。
> 主要记录了我在实现炮管上下旋转限制时遇到的问题，以及对 Unity EulerAngles / localEulerAngles 角度机制的理解过程。

------

# 1. Clamp 为什么没有效果

### 我的问题

代码：

```csharp
Math.Clamp(gun.transform.rotation.eulerAngles.x, -20, 20);
gun.Rotate(new Vector3(inputPitch.y,0,0));
```

为什么限制没有生效。

------

### GPT 告诉我的内容

Clamp 只是：

```text
计算一个结果
```

不会自动修改 rotation。

因此：

```csharp
Math.Clamp(...)
```

如果不赋值：

```csharp
x = Math.Clamp(...)
```

就等于没做任何事。

另外：

```csharp
Rotate()
```

会继续累加旋转。

即使 Clamp 正确，也会被 Rotate 绕过去。

因此：

- 不适合用 Rotate 做炮管俯仰
- 更适合直接控制 localEulerAngles

------

# 2. Unity 欧拉角范围问题

### 我的问题

我问：

```text
localEulerAngles 也是 0-360 对吗？
```

因为 Clamp 的：

```text
[-20,20]
```

似乎无法工作。

------

### GPT 告诉我的内容

Unity 的：

```csharp
eulerAngles
localEulerAngles
```

范围都是：

```text
0~360
```

不会返回负数。

例如：

| 实际角度 | Unity返回 |
| -------- | --------- |
| -1°      | 359       |
| -10°     | 350       |
| -20°     | 340       |

因此：

```csharp
Mathf.Clamp(x,-20,20)
```

直接使用会失效。

------

# 3. “假负数”理解过程

### 我的疑问

我一开始以为：

```text
既然初始值是 0
那不是永远无法进入：
if(x > 180)
```

------

### GPT 告诉我的内容

Unity 在往负方向旋转时：

不会：

```text
0 → -1 → -2
```

而是：

```text
0 → 359 → 358
```

因此：

```csharp
if (x > 180)
    x -= 360;
```

实际上是在做：

```text
359 -> -1
350 -> -10
340 -> -20
```

也就是：

把 Unity 的：

```text
[0,360]
```

转换成：

```text
[-180,180]
```

这样 Clamp 才真正可用。

------

# 4. 最终理解

我最后理解到：

```text
Unity 并不存在真正的负欧拉角
所谓负角度其实是：
359
350
340
```

因此：

```csharp
if (x > 180)
    x -= 360;
```

本质上是：

```text
把“伪负数”恢复成真正的负数
```

这是 Unity 中：

- 摄像机 Pitch
- 坦克炮管俯仰
- 飞机俯仰

最经典的处理方式之一。
